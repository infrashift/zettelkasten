INTRODUCTION                                                    *zk* *zk.nvim*

zk.nvim provides NeoVim integration for the zk CLI tool, enabling seamless
management of a Zettelkasten note-taking system directly from your editor.

Features:
  - Create untethered and tethered notes
  - Daily notes with review workflows
  - Note templates (meeting, user-story, feature, etc.)
  - Full-text search with snacks.nvim picker integration
  - Graph visualization of note relationships
  - Backlinks discovery
  - Tag completion in YAML frontmatter
  - Link insertion between notes

REQUIREMENTS                                                 *zk-requirements*

  - NeoVim 0.9+
  - plenary.nvim (required)
  - snacks.nvim (optional, for picker UI)
  - zk CLI binary in $PATH

INSTALLATION                                                 *zk-installation*

Using lazy.nvim: >lua
    {
        "infrashift/zettelkasten-cli",
        dependencies = {
            "nvim-lua/plenary.nvim",
            "folke/snacks.nvim",
        },
        config = function()
            require("zk").setup({
                bin = "zk",
            })
        end,
    }
<

Using packer.nvim: >lua
    use {
        "infrashift/zettelkasten-cli",
        requires = { "nvim-lua/plenary.nvim" },
        config = function()
            require("zk").setup()
        end,
    }
<

SETUP                                                               *zk-setup*

                                                              *zk.setup()*
require("zk").setup({opts})

    Configure the plugin. Call this in your init.lua or plugin config.

    Options: ~
        {bin}    (string)  Path to zk binary. Default: "zk"

    Example: >lua
        require("zk").setup({
            bin = "/usr/local/bin/zk",
        })
<

COMMANDS                                                         *zk-commands*

All commands are prefixed with "Zk" for discoverability.

Daily Notes ~
                                                                    *:ZkDaily*
:ZkDaily [date]         Open or create a daily note.
                        - No argument: today's note
                        - "yesterday": yesterday's note
                        - YYYY-MM-DD: specific date

                                                                *:ZkDailyList*
:ZkDailyList[!]         Browse daily notes with picker.
                        With !: show this week only

Note Creation ~
                                                                      *:ZkNote*
:ZkNote [category] [project]
                        Create a new zettel.
                        {category}: "untethered" (default) or "tethered"
                        {project}: optional project name

                                                                 *:ZkTemplate*
:ZkTemplate [name]      Create note from template.
                        Without argument: opens template picker

Search ~
                                                                   *:ZkSearch*
:ZkSearch[!] [flags] [query]
                        Search zettels with optional filters.
                        With !: live search (updates as you type)

                        Flags: ~
                          --category {val}   untethered or tethered
                          --project {val}    filter by project name
                          --type {val}       note, todo, daily-note, issue
                          --status {val}     open, in_progress, closed
                          --priority {val}   high, medium, low
                          --due-before {val} YYYY-MM-DD
                          --due-after {val}  YYYY-MM-DD
                          --tag {val}        filter by tag (repeatable)

                        Examples: >vim
                          :ZkSearch authentication
                          :ZkSearch --category tethered
                          :ZkSearch --type todo --status open
                          :ZkSearch --project my-project --priority high
                          :ZkSearch! --type todo
<

Indexing ~
                                                                    *:ZkIndex*
:ZkIndex [path]         Index zettels for searching.
                        Default: current directory

Graph ~
                                                                    *:ZkGraph*
:ZkGraph [args]         Show graph in a scratch buffer (Mermaid syntax).
                        {limit}: max nodes (default: 10)
                        --depth N: max BFS hops
                        --start ID: center on a zettel
                        Press q to close the buffer.

                        Output is Mermaid flowchart syntax: >
                          graph LR;
                            A["Note Title A"];
                            B["Note Title B"];
                            A --> B;
<
                        Nodes are declared with their titles as labels.
                        A --> B means A has [[B]] in its body.
                        Paste into mermaid.live or any Mermaid renderer.

                                                                   *:ZkExport*
:ZkExport [args]        Export graph subset as portable markdown.
                        --limit N: max notes to export (default: 10)
                        --depth N: max BFS hops
                        --start ID: center on a zettel

                        Copies notes into <root>/ephemeral/ with
                        [[id|title]] links converted to [title](id.md)
                        relative links. The ephemeral/ directory has a
                        .gitignore that ignores all exported files.

                        Links to notes NOT in the export set are left
                        unchanged. The directory is wiped on each export.

Tags ~
                                                              *:ZkRefreshTags*
:ZkRefreshTags          Refresh tag cache.

LUA API                                                             *zk-api*

All functions are available via `require("zk")`.

Daily Notes ~
                                                                  *zk.daily()*
daily({opts})           Open or create daily note.
                        opts.date: "yesterday" or "YYYY-MM-DD"

                                                            *zk.daily_picker()*
daily_picker({opts})    Browse daily notes with picker.
                        opts.week: show this week only
                        opts.month: show this month only

                                                             *zk.list_daily()*
list_daily({opts}, {callback})
                        Get daily notes asynchronously.

                                                        *zk.list_daily_sync()*
list_daily_sync({opts}) Get daily notes synchronously.

Note Creation ~
                                                            *zk.create_note()*
create_note({category}, {project})
                        Create a new zettel.

                                                     *zk.create_from_template()*
create_from_template({opts})
                        Create note from template.
                        opts.template: template name
                        opts.project: project name

                                                         *zk.template_picker()*
template_picker({opts}) Open picker for templates.

Note Management ~
                                                            *zk.tether_note()*
tether_note({path}, {project})
                        Tether a note (set category to tethered).

                                                          *zk.untether_note()*
untether_note({path})   Untether a note (set category to untethered).

                                                            *zk.set_project()*
set_project({path}, {project})
                        Set project for a zettel.

Search ~
                                                                 *zk.search()*
search({query}, {opts}) Search zettels.
                        opts.project, opts.category, opts.tags, opts.limit
                        opts.type, opts.status, opts.priority
                        opts.due_before, opts.due_after
                        opts.on_results: callback function

                                                                  *zk.index()*
index({path})           Index zettels.

Graph ~
                                                                  *zk.graph()*
graph({opts})           Show Mermaid graph in scratch buffer.
                        opts.path, opts.limit, opts.depth, opts.start

                                                                 *zk.export()*
export({opts})          Export graph subset as portable markdown.
                        opts.path, opts.limit, opts.depth, opts.start

                        from [[id|title]] to [title](id.md).

Preview ~
                                                           *zk.preview_note()*
preview_note({path})    Preview note in floating window.

                                                          *zk.preview_by_id()*
preview_by_id({id})     Preview note by ID.

Linking ~
                                                            *zk.insert_link()*
insert_link({id}, {title}, {include_title})
                        Insert link at cursor.

                                                            *zk.link_picker()*
link_picker({opts})     Open picker for linking.
                        opts.include_title: use [[id|title]] format

Backlinks ~
                                                          *zk.get_backlinks()*
get_backlinks({id}, {callback})
                        Get backlinks asynchronously.

                                                        *zk.backlinks_panel()*
backlinks_panel({opts}) Open floating backlinks panel.

                                                        *zk.backlinks_split()*
backlinks_split({opts}) Open backlinks in split.

toggle_backlinks()      Toggle backlinks panel.

Tags ~

validate({path})        Validate frontmatter against CUE schema.
                        {path}: optional file path (default: current buffer)

                                                               *zk.get_tags()*
get_tags({callback})    Get all tags asynchronously.

                                                          *zk.get_tags_sync()*
get_tags_sync()         Get all tags synchronously.

                                                          *zk.refresh_tags()*
refresh_tags()          Clear and reload tag cache.

                                                         *zk.complete_tags()*
complete_tags()         Trigger tag completion at cursor.

                                                   *zk.setup_tag_completion()*
setup_tag_completion()  Set up tag completion for markdown.

                                                              *zk.setup_cmp()*

PICKER                                                           *zk-picker*

If snacks.nvim is installed, additional functions are available:

require("zk.picker").search({opts})      Search all zettels
require("zk.picker").live_search({opts}) Live search

    Search opts: ~
        {project}     Filter by project name
        {category}    Filter by category (untethered/tethered)
        {type}        Filter by type (note/todo/daily-note/issue)
        {status}      Filter by status (open/in_progress/closed)
        {priority}    Filter by priority (high/medium/low)
        {due_before}  Filter due before date (YYYY-MM-DD)
        {due_after}   Filter due after date (YYYY-MM-DD)
        {tags}        Table of tag strings (AND logic)
require("zk.picker").untethered({opts})  Browse untethered notes
require("zk.picker").tethered({opts})    Browse tethered notes
require("zk.picker").insert_link({opts}) Insert link to zettel

Picker Keymaps ~
    <CR>        Open note
    <C-p>       Preview in floating window
    <C-l>       Insert link as [[id]]
    <C-S-l>     Insert link as [[id|title]]

FILETYPE SETTINGS                                           *zk-ftplugin*

When editing a zettel file (markdown with id: in frontmatter), the plugin
automatically sets up:

  - omnifunc for tag completion
  - <C-x><C-t> for manual tag completion
  - <localleader>l for link insertion
  - <localleader>L for link insertion with title
  - <localleader>b for backlinks toggle
  - <localleader>a for adding tags (all zettel types)
  - <localleader>v for validating frontmatter (all zettel types)
  - <localleader>p for set project (note and todo types)
  - <localleader>t for tether/untether picker (note and todo types)
  - <localleader>s for todo status picker (todo-type only)

SUGGESTED KEYMAPS                                               *zk-keymaps*

Add these to your configuration: >lua

    -- Daily notes
    vim.keymap.set("n", "<leader>zd", "<cmd>ZkDaily<cr>")
    vim.keymap.set("n", "<leader>zD", "<cmd>ZkDaily yesterday<cr>")
    vim.keymap.set("n", "<leader>zw", "<cmd>ZkDailyList<cr>")

    -- Note creation
    vim.keymap.set("n", "<leader>zf", function()
        require("zk").create_note("untethered")
    end)
    vim.keymap.set("n", "<leader>zp", function()
        require("zk").create_note("tethered")
    end)
    vim.keymap.set("n", "<leader>zt", "<cmd>ZkTemplate<cr>")

    -- Search
    vim.keymap.set("n", "<leader>zz", "<cmd>ZkSearch<cr>")
    vim.keymap.set("n", "<leader>z/", "<cmd>ZkSearch!<cr>")

    -- Graph
    vim.keymap.set("n", "<leader>zg", "<cmd>ZkGraph<cr>")
<
